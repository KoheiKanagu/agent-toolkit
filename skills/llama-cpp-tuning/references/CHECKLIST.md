# llama.cpp チューニング チェックリスト

## 事前準備

- [ ] 対象モデルが決まっている
- [ ] llama-bench / llama-server が使用可能
- [ ] ベンチマーク用のプロンプトを準備（短いもの + 長いもの）
- [ ] 結果を記録する場所を用意

## フェーズ1: 環境確認

- [ ] CPU物理コア数を確認
- [ ] メモリ総量を確認
- [ ] GPU情報を確認
- [ ] ツールバージョンを確認

## フェーズ2: ベースライン

- [ ] `llama-bench -hf <model> -o md` を実行
- [ ] pp512 / tg128 を記録

## フェーズ3: パラメータスイープ

### スレッド数

- [ ] `-t 4,6,8,10`（または物理コア数周辺）で計測
- [ ] 最適値を決定

### バッチサイズ

- [ ] `-b 512,1024,2048,4096` で計測
- [ ] `-ub 128,256,512,1024,2048` で計測
- [ ] 最適値を決定

### Flash Attention

- [ ] `-fa auto,on,off` で計測
- [ ] 最適値を決定

### GPUレイヤー

- [ ] `-ngl -1` と `-ngl 0` を比較
- [ ] 通常は `-ngl -1` を採用

### キャッシュタイプ

- [ ] `-ctk f16,q8_0 -ctv f16,q8_0` を試す
- [ ] 失敗する場合は f16 を維持

## フェーズ4: コーディングエージェント用途

- [ ] 長文プロンプト（2K/4K/8K）で計測
- [ ] llama-server を起動
- [ ] API経由で実レイテンシを計測
- [ ] プロンプトキャッシュあり/なしを比較

## フェーズ5: まとめ

- [ ] 最適パラメータを決定
- [ ] 推奨コマンドを作成
- [ ] 結果を記録
- [ ] 注意事項を記録
